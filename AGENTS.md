# AGENTS.md

> For AI coding assistants. Human developers: see README.md first.

## Critical Rules

- **NEVER edit `docs/` directly** — all files are auto-generated by `scripts/build-docs.py`
- **ONLY edit `README.md`** for content changes — the build script parses it
- Run `python3 scripts/build-docs.py` after any README.md change to regenerate docs

## Project Structure

```
README.md              # Source of truth — only file to edit for content
scripts/build-docs.py  # Build script (Python 3.11+, no deps)
docs/                  # Generated static site — DO NOT EDIT
static/img/            # Images referenced by README.md
```

## Build & Verify

```bash
# Build docs site
python3 scripts/build-docs.py

# Preview locally
cd docs/ && python3 -m http.server 8000

# Check broken links (requires lychee)
lychee --accept 100..=399 --timeout 30 README.md
```

## Content Format

Tool entries in README.md use this exact pattern:

```html
&nbsp;&nbsp; <a href="URL"><b>Tool Name</b></a> - Description.<br>
```

## CI Behavior

- `build-docs.yml`: Auto-runs on master push (README.md, scripts/, docs/css/, docs/js/), commits generated docs
- `check-links.yml`: Monthly + manual + README.md push; uses lychee with excludes for shields.io, opencollective.com, github.com
